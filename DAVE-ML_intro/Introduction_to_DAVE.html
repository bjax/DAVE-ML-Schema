Introduction to DAVE-ML

 

021108 Bruce Jackson e.b.jackson@larc.nasa.gov

 

Mission

To provide a digital exchange format for aerospace vehicle flight dynamic models in a facility- and programming-language-neutral way, using open standards. The ultimate goal is drastically reduce the time required to move an operating, real-time simulation model from one simulation laboratory to another in less than one work day.

Introduction

This document describes an initial attempt to develop a text-based markup language,  using the eXtensible Markup Language (XML) syntax,  to use to exchange flight simulation data. This XML "application" is formally the Dynamic Aerospace Vehicle Exchange Markup Language, DAVE-ML.  The concept was developed and demonstrated in the summer of 2002 by Bruce Jackson of NASA Langley Research Center for application to some projects he was working on there. It is based on simulation data standards developed by the AIAA Modeling and Simulation Technical Committee under the leadership of Bruce Hildreth of SAIC.

A paper regarding the DAVE-ML concept was presented at the 2002 AIAA Modeling and Simulation Technologies conference by Jackson and Hildreth (see the reference at the bottom of this document). An estimated savings of $7 million per year for a single in-service tactical aircraft program is identified, based on simulation model configuration control and rapid error correction dissemination alone.

XML is a formal standard supported by the World-Wide Web Consortium (W3C). More information may be found at http://www.xml.org/.

Required XML applications

To take advantage of existing XML developments, DAVE relies on the Mathematics Markup Language (MATH-ML, version 2.0). More information on this XML application can be found at http://www.w3.org/Math/.

Status

At present, only a portion of the markup required to describe a full-blown non-linear simulation, complete with checkcases, trim shots, and test input descriptions, has been designed. The current implementation focuses on the bulk of any flight vehicle model, namely the data tables.  To this end, a Data Type Description (DTD) has been developed to encode documentation references, sets of tables, breakpoints, and the buildup equations to link these together with other input variables to form output relationships (e.g. coefficients, forces and moments).

The DAVEfunc.dtd file, described below, contains the markup elements for this part of the full DAVE-ML application. Missing from the design is support for dynamic elements, so control laws higher than type 0 are not yet supported. Also, full implementation of the draft AIAA aero table format is not yet available since the DTD does not yet implement statistics for individual data points.

A Java application has been developed that parses DAVEfunc XML files and generates Simulink release 11 (and higher) .mdl files that can be exercised in Matlab; see DAVE2SL below.

DAVEfunc.dtd description

The latest version of DAVEfunc.dtd should be included with this package. If not, visit the DAVE-ML website at http://dcb.larc.nasa.gov/utils/fltsim/DAVEor contact the author.

 

The DAVEfunc Data Type Definition serves as the roadmap for constructing a compliant DAVE-ML function description. It should be sufficient to develop any static model but is designed to work for n-dimensional tables (and eventually polynomial functions).  Elements of DAVEfunc include references to external documents, constants, variables, breakpoint sets, multidimensional tables, and mathematical equations. DAVEfunc elements describe basically any static mathematical relationship (via use of MATH-ML) between input and output variables, allow infinite dimensionality in the data tables, and support statistics and provenance information about any table or single data point.

 

A brief outline of the available elements is given below. Refer to the DAVEfunc.dtd file for more exact information, or take a look at the examples for clarification.

Examples

Simple table

The simplest DAVEfunc description is a single one-dimensional table, shown below. Some of the file header content could be omitted while retaining a valid DAVEfunc document. No input or output variables have been defined, so this would not be renderable into Simulink with the DAVE2SL utility.

 

<?xml version="1.0" standalone="no"?>

<!DOCTYPE DAVEfunc SYSTEM "DAVEfunc.dtd">

<DAVEfunc>

    <fileHeader>

        <author name="Bruce Jackson" org="NASA Langley Research Center" xns="@bjax"/>

        <fileCreationDate date="11-MAR-2002"/>

        <description>

               Coefficient of lift (non-dimensional) versus angle-of-attack, deg.

               Example file for DAVE function table format. This example is the simplest version.

           </description>

    </fileHeader>

    <function name="CL">

           <independentVarPts>

             -4.0, 0., 4.0, 8.0, 12.0, 16.0

           </independentVarPts>

        <dependentVarPts>

             0.0, 0.2, 0.4, 0.8, 1.0, 1.2

        </dependentVarPts>

    </function>

</DAVEfunc>

F-16 nonlinear model

Included in this package should be a complete F-16 non-linear subsonic aerodynamics model as described in the Stevens & Lewis, based on a NASA wind tunnel experiment. This data was compiled into a Matlab script by Gene Morelli of NASA Langley for an upcoming technical memorandum.  The DAVE2SL utility has been used to generate a Simulink realization from the DAVE-ML file, and compared identically to the Matlab version by Morelli.

dave2sl utility

Included in this package is a folder containing the source code for a Java 1.3 proof-of-concept demonstration that parses a DAVEfunc XML file and generates an equivalent Simulink model. No warranty is expressed or implied for this utility that has not been fully tested. It requires additional Java source code, including xerces-2 and jdom-b8, as well as the mathml2 data type definition.

Related work

Jon Berndt jsb@hal-pc.org has written, as part of his JSBsim shareware flight simulation code, an XML-based model description specification, which served as an impetus for this work. More information on this specification is available at http://jsbsim.sf.net/.

Bruce Hildreth bruce@pax.saic.com has worked with the AIAA Modeling and Simulation Technical committee to develop draft standards for function table data, variable names, and time history data files. This effort (DAVE) is intended to follow these guidelines to the maximum extent possible.
